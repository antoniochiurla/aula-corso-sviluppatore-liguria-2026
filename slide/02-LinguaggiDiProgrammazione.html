Linguaggio - Come "pensa" il programmatore

Assembly- Pensa ai circuiti (muovi bit tra i registri).
RPG/COBOL - Pensa ai moduli cartacei (compila colonne e sezioni).
C/C++ - Pensa alla macchina (gestisci memoria e tipi).
Java - Pensa agli oggetti (incapsula tutto in classi).
Python - Pensa all'algoritmo (scrivi la logica nel modo più naturale).

string - tipi di dati

  • il tipo string contiene sequenze di caratteri
    • la lunghezza massima dipende dalla memoria disponibile
    • si possono ottenere porzioni di stringa
    • si possono concatenare con l'operatore +
    • vanno racchiuse tra virgolette, singole o doppie a scelta
      • il carattere virgoletta usato per racchiuderlo va preceduto da \
        • Esempio: 'Ha detto \'Ahi\', quando si è fatto male."
    • formattazione
      • f'Valore della variabile value: {value}"'

int - tipi di dati

  • il tipo int contiene numeri interi
    • i valori possono essere positivi e negativi o 0 (zero)
    • rappresentazione interna è binaria, una serie di bit
    • Python 3, a differenza di altri linguaggi, non ha limiti di valore, se non la dimensione di memoria
    • Python 2 sys.maxint su CPU a 64 bit dovrebbe essere 2**63-1
    • i numeri interi si possono
      • + sommare, - sottrarre, * moltiplicare, / dividere, // divisione intera
      • % resto della divisione, ** elevare a potenza
    • 1 2 +3 -5 +327888943

float - tipi di dati

  • il tipo float contiene numeri decimali
    • rappresentazione interna in virgola mobile
      • <segno><coefficiente><esponente>
      • <segno> rappresentato da un bit
        • 1 negativo
        • 0 positivo
      • <coefficiente> serie di bit
      • <esponente> serie di bit che rappresentano l'esponente della potenza di 2 da moltiplicare al <coefficiente>
    • limitazioni: https://docs.python.org/it/3.13/tutorial/floatingpoint.html
    • alternativa: Decimal per trattare numeri estremamente grandi o piccoli

Conversioni

  • Le conversioni
    • cambiano tipo creando una nuova rappresentazione del valore
    • da string a int
      • numero = int('42') # si ottiene 42
      • numero = int('42x3') # si ottiene ValueError: invalid literal for int() with base 10: '42x3'
      • numero = int('52',8) # si ottiene 42, interpretando 52 in base 8
      • numero = int('2A',16) # si ottiene 42, usando la base 16
    • da int a string
      • stringa = str(42) # si ottiene '42'
    • da int a float
      • decimale = float(42) # si ottiene 42.0
    • da float a int
      • risposta = int(42.7) # si ottiene 42 come la parte intera di 42.7

Il bivio: if, elif, else

Esempio: 

eta = 18
if eta > 18:
	print("Puoi votare")
elif eta == 18:
	printf("Forse è la prima volta che voterai")
else:
	print("Troppo giovane")

Importante: l'indentazione (lo spazio a inizio riga) è importante

Confrontare i Valori

  • Operatori di confronto:
    • == (uguale a)
    • != (diverso da)
    • > e < (maggiore/minore)
  • Operatori logici (in ordine di precedenza)
    • NOT
    • AND
    • OR
  • Il risultato è sempre un Booleano (True o False).

Concetti OOP: classi, oggetti, metodi

  • Classe: È il progetto, il "blueprint". Definisce quali dati (attributi) e quali azioni (metodi) avrà un oggetto.
    • Esempio TaMaSy: La classe Task in 01-Task-Class.py definisce che ogni compito deve avere un titolo, una descrizione e una priorità.
  • Oggetto (Istanza): È la realizzazione concreta della classe.
    • Esempio TaMaSy: task1 ("Comprare il latte") e task2 ("Studiare Python") sono due oggetti diversi nati dalla stessa classe.
  • Metodi: Sono le funzioni "dentro" la classe che definiscono il comportamento dell'oggetto.
    • Esempio TaMaSy: Il metodo .completa() che cambia lo stato del task da "aperto" a "chiuso".

OOP: costruttori, incapsulamento

  • Costruttore (__init__ in Python): Il metodo speciale che viene chiamato quando si crea l'oggetto. Serve a "inizializzare" i dati (es. assegnare il titolo al task appena nato).
  • Incapsulamento: Nascondere i dettagli interni dell'oggetto e mostrare solo ciò che serve all'esterno. Si usano i "livelli di accesso" (pubblico/privato).
    • Concetto Pou: Non sappiamo come Pou digerisce il cibo (codice interno), vediamo solo che la barra della fame sale (interfaccia pubblica).

Gestione delle eccezioni e degli errori runtime.

  • Quando qualcosa non funziona come dovrebbe
    • Errori Runtime
      • Errori che capitano mentre il programma è acceso (es. dividere per zero o cercare un file che non esiste).
    • Eccezioni (try...except): Un modo per "catturare" l'errore prima che faccia crashare il programma.
      • Esempio TaMaSy in 03-Task-Exception.py: Se l'utente inserisce una priorità non valida, il programma non deve chiudersi, ma dire: "Ehi, priorità non valida!".

Ereditarietà, polimorfismo

  • Ereditarietà: Una classe "figlia" prende tutto dalla classe "padre" e aggiunge qualcosa di specifico.
    • Esempio TaMaSy: La classe TaskUrgent eredita da Task ma aggiunge un segnale acustico o un colore rosso.
  • Polimorfismo: La capacità di oggetti diversi di rispondere allo stesso comando in modi diversi.
    • Esempio TaMaSy: Se chiamo il metodo .notifica(), un task normale invia una mail, un task urgente invia un SMS. Il comando è lo stesso, il risultato cambia.

Overriding e overloading.

  • Overriding: Quando la classe figlia riscrive un metodo del padre perché deve fare qualcosa di diverso.
    • Esempio TaMaSy: TaskUrgent sovrascrive il metodo .mostra() per aggiungere dei punti esclamativi al titolo.
  • Overloading: Avere più metodi con lo stesso nome ma che accettano parametri diversi (es. calcola(x) e calcola(x, y)). Nota: In Python puro non esiste come in Java/C#, si gestisce con i parametri opzionali.
  • In 04-Task-OOP.py la classe Task e le derivate BugTask e FeatureTask

UseCase - UML

  • L'Attore (L'omino)
    • L'Attore non è "una persona", ma un ruolo. Nel nostro caso è l'utente che usa TaMaSy per gestire i suoi impegni
  • Il Confine del Sistema (Il rettangolo)
    • Tutto ciò che sta dentro il rettangolo è il software che stiamo scrivendo (TaMaSy)
    • Quello che sta fuori è il mondo esterno (l'utente)
  • Il concetto di <<include>>
    • Le frecce tratteggiate. Ogni volta che l'utente modifica qualcosa (Aggiunta, Rimozione, Stato), l'azione include sempre il "Salvataggio su File"
    • È un modo per dire: "Non puoi fare una modifica senza che il sistema la salvi"
