Test Funzionale

  • Verificare che l'intero flusso (dalla UI al Database) rispetti i requisiti

Test di Compatibilità

  • TaMaSy appare bene su
    • Chrome
    • Firefox
    • Safari
    • ... e su mobile

Test di Regressione

  • Fondamentale
  • Assicurarsi che l'integrazione di una nuova funzione non abbia rotto una funzione vecchia

Smoke test

  • Test del fumo
    • ad ogni build verificare
      • che "sta in piedi"
      • che non ci sono errori macroscopici

Test di Performance

  • Misurare quanto tempo impiega una pagina a caricarsi
  • Quanto impiega ad avviarsi
  • Quanto impiega una specifica operazione impegnativa

TaMaSy

  • Il codice è integrato
  • Ma l'app è ancora fluida?
  • Facciamo un check completo

Strumenti per Test di Integrazione

  • PyTest
    • Per testare il dialogo tra i componenti (es. la View di Django parla correttamente col Modello?).
  • Postman Collections
    • Fondamentale per testare le API
  • Selenium o Playwrigth o JMeter
    • Simula un utente reale che clicca sui bottoni del browser
    • Il "test robotizzato" dell'interfaccia
  • TaMaSy
    • Simulare 10 inserimenti di task consecutivi e vedere se il sistema regge

Analisi del Carico e Stress Test

  • Load Test
    • Verificare il comportamento sotto il carico previsto (es. 100 utenti contemporanei)
  • Stress Test
    • Spingere il sistema al limite finché non si rompe (es. 10.000 utenti)
    • Obiettivo: capire il "punto di rottura"
  • Benchmarking
    • Confrontare le prestazioni hardware (CPU/RAM usata) e software (tempo di risposta in millisecondi)
  • TaMaSy
    • Cosa succede a TaMaSy se tutta la classe preme 'Salva' nello stesso istante?

Benchmarking delle prestazioni

  • Attività di osservazione e raffronto dei propri processi con parametri di riferimento (benchmark)
  • Obiettivi
    • migliorare la competitività
    • identificare gap prestazionali
    • ottimizzare il software
  • Processo
    • si sviluppa attraverso
      • la pianificazione
      • la raccolta dati
      • l'analisi per individuare le differenze
      • la pianificazione delle azioni correttive

Analisi Log

  • Analisi Log
    • Imparare a leggere i file di log del server (es. log di Django o del database) per trovare errori silenziosi
  • Tracciamento errori
    • Uso di strumenti per intercettare i crash
  • TaMaSy
    • Riconoscere un malfunzionamento leggendo i log

Ottimizzazione del sistema

  • Ottimizzazione
    • Se una query SQL è lenta, come la miglioriamo?
      • Esempio: aggiungere un Indice al Database o usare la cache
  • TaMaSy
    • Abbiamo trovato una query lenta che rallenta la Dashboard: ottimizziamola!

Gestione Rischi e Monitoraggio

  • Procedure di Rollback
    • Se lanciamo un aggiornamento di TaMaSy e tutto si rompe, come torniamo alla versione precedente in 60 secondi?
    • L'importanza dei backup e di Git

Monitoraggio & Alerting:

  • Prometheus
    • Raccoglie dati sulle performance
  • Grafana
    • Crea cruscotti (dashboard) bellissimi con grafici sull'uso di CPU e tempi di risposta
  • TaMaSy
    • Costruiamo un piccolo pannello di controllo per vedere se il server di TaMaSy è vivo o morto
